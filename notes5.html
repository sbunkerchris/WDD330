Preparation Notes

Errors, Exeptions, Warnings

  • System Error - Problem with system or other attached devices
  • programmer error - program is faulty
  • user error - user input data and the program cant handle it
  • >Exception - the message that is "thrown" before the program terminate

Strict mode

  • a mode of javascript (ECMAScript 5) that produces more warnings and exceptions and prohibits use of some deprecated features.
  • using "strict" in the beginning of a file will enforce this mode.

Linting Tools

  • tools that can assist (and test) code quality
  • highlights sloppy code
  • identifies syntax errors
  • style conventions are not followed

Feature Detection

  • API changes often and also is dependant on browser, updates, users...and many other factors.
  • Feature Detection - the way of identifying the users browsers applicable features.
  • uses an if statement to run the code (or method) if that code exists. this can prevent errors.

Debugging

  • Debugging - process of finding bugs and where they are in the code.
  • breakpoints - areas in the code that stop the progress so view the value of variables at that point in the program.
  • alert(value) - function/method that displays a message on screen (with an ok button) will also display the value in the dialogue box of the passed value.
  • most modern browsers have these tools built in

Error Objects

  • Error(string) - (constructor) if a string is put in it
  • RangeError - when a number is outside allowable range of values
  • ReferenceError - reference is made to something that doesnt exist
  • SyntaxError - For sytax errors...duh
  • TypeError - value type does not match
  • >URIError - Problem encoding or decoding URI
  • URI - string of characters identifying a resource (most common is a URL)

Exception Handling

  • catch, try
  • from a personal view this is one of the factors that makes usability for both user and programmer.
